
-------------------- BACKEND --------------

npm e yarn são gerenciadores de pacote

"yarn init -y "  na pasta do projeto 


package.JSON - guarda características do projeto que usa javascript

JSON também é uma estrutura de dados utilizada para fazer a comunicação entre o front-end e o back-end

node_modules NÃO DEVE SER ALTERADO

CTRL+C = PARA A EXECUÇÃO

"yarn add express" na pasta do projeto 

EXPRESS  - microframework

lida com as rotas






"node index.js" 'executa' o código

"yarn add nodemon -D " na pasta do projeto 
	instala uma dependencia de desenvolvimento, para, ao salvar o codigo no vscode, atualizar no navegador
	( ou seja, faz com que o node continue obtendo as informações do arquivo mesmo que ja tenha sido executado)

"yarn nodemon index.js" executa o projeto com o nodemon

 - criar um script no package.json 
	"scripts": {
    		"dev": "nodemon index.js"
  	},
	
	com isso: "yarn dev" executará o projeto

//METODO DO EXPRESS (HTTTP) 
// get - busca informação  ex. buscar um usuario
// post - criar informação ex. cadastrar um usuário
// put - editar uma informação 
// delete - deletar uma informação

//TIPOS DE PARÂMETRO
//Query Params:  request.query (Filtros, ordenação, paginação, ...) MAIS USADO NO GET
//Route Params:  request.params (Indetificar um recurso na alteração ou remoção)        MAIS USADO NO PUT E DELETE
//Body:  request.body (Dados para a criação ou alteração de registro)




MONGO DB (banco de dados não-relacional)
